QS Vault: Professional Modular Architecture

To ease debugging and mirror the professional Quantity Surveying workflow, we are splitting the current monolithic files into logical sub-modules. This structure follows the SMM-KE (Standard Method of Measurement) hierarchy.

1. Global Layout Modules

SidebarCommand.tsx: Handles primary node navigation (Registry, Rates, Settings).

HUDHeader.tsx: Provides session metadata, connectivity status, and the Identity Dropdown.

2. Dashboard Sub-Components

StatGrid.tsx: Real-time infrastructure metrics (Vaults, Takeoffs, Node Health).

VaultRegistry.tsx: The master project inventory.

RatesLibrary.tsx: The regional resource database (Material, Labor, Plant).

IdentityNode.tsx: Profile and security clearance management.

3. Takeoff Engine (Project Details)

CalibrationNode.tsx: Dedicated scale and unit setting (Crucial for precision).

BlueprintViewport.tsx: The PDF rendering and SVG measurement overlay logic.

GeometricRegistry.tsx: The "Stream" of raw measurements captured from the site node.

SMMWorkSections.tsx: Tool selection and section mapping (Excavations, Concrete, etc.).

4. Billing & Analytics

BoQGenerator.tsx: Translates raw takeoff dimensions into priced Bill items.

ArtifactsVault.tsx: PDF/Excel generation for Interim Payment Certificates (IPC) and Elemental Estimates.

5. Professional QS Workflow Enhancements

Audit Handshake: Every measurement now links back to a specific coordinate on the drawing.

Waste Factor Overrides: Ability to add percentage-based waste per material category.

Valuation Delta: Comparing "Measured Quantities" vs "Contract Quantities" for Certifications.
